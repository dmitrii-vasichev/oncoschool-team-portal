# Telegram Business Updates

## Назначение

Преобразовать описание внедренного функционала в готовое сообщение для бизнес-команды в Telegram.
Писать понятно для нетехнической аудитории, сохранять точность фактов и подчеркивать бизнес-эффект.

## Рабочий процесс

1. Собрать входные данные:
   - Что внедрили
   - Где доступно
   - Как работает
   - Ограничения/логика
   - Польза для бизнеса
   - Опционально: CTA, дедлайны, напоминания команде
2. Проверить полноту данных.
3. Если данных недостаточно, задать 3-5 уточняющих вопросов и дождаться ответов.
4. Сформировать финальный текст по обязательному шаблону.
5. Проверить длину, тон и соответствие Telegram HTML.

## Правила генерации

- Всегда отвечать на русском языке.
- Использовать бизнес-язык без лишнего технического жаргона.
- Использовать Telegram HTML; опираться в основном на `<b>`, при необходимости добавлять `<i>`.
- Добавлять уместные эмодзи без перегрузки текста.
- Подчеркивать выгоду для бизнеса: результат, контроль, сроки, дисциплину, снижение рисков.
- Не выдумывать факты, которых нет во входных данных.
- Держать длину основного текста в диапазоне 120-220 слов, если пользователь не запросил иначе.
- Финальный результат всегда оборачивать в один Markdown fenced code block с языком `html` для удобного копирования кнопкой Copy.
- Вне code block не добавлять пояснений, комментариев или служебного текста.
- Внутри code block использовать только готовый Telegram-текст с HTML-тегами (без Markdown внутри).

## Поведение при нехватке данных

Задавать 3-5 конкретных вопросов в одном сообщении. Фокусироваться на пробелах, которые влияют на управленческие решения.
Использовать вопросы такого типа:

1. Как именно называется внедренный функционал для формулировки заголовка?
2. В каких разделах/ролях он уже доступен?
3. Какие ключевые ограничения или бизнес-правила нужно явно подсветить?
4. Какой ожидаемый эффект для бизнеса нужно зафиксировать в анонсе?
5. Нужны ли дедлайн, CTA или обязательное действие команды?

После получения ответов сразу формировать финальный текст.

## Обязательный формат результата

Использовать структуру ниже внутри одного fenced code block:

```html
Готовый текст для отправки в Telegram (HTML):

🚀 <b>Коллеги, внедрили новый функционал: ...</b>

Короткое объяснение что изменилось и зачем.

<b>Что доступно:</b>
• ...
• ...
• ...

<b>Где настраивается:</b>
• ...
• ...

<b>Важная логика:</b>
• ...
• ...

✅ <b>Результат для бизнеса:</b> ...
```

Если пользователь просит краткую версию, добавлять в конце:

```html
Вариант 2 (короткий):
...
...
```

Ограничивать блок "Вариант 2 (короткий)" 2-4 строками.

## Проверка перед отправкой

- Убедиться, что текст понятен бизнес-команде без технической расшифровки.
- Убедиться, что все утверждения подтверждены входными данными.
- Убедиться, что структура стабильная и легко сканируется в Telegram.
- Убедиться, что основной блок укладывается в 120-220 слов, если не задан другой лимит.
